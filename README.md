## б) Постановка задачи
Разработка программной модели социальной сети на языке программирования Python с применением принципов объектно-ориентированного программирования (ООП) для демонстрации навыков проектирования классовых структур, работы с файловыми системами и создания интерактивного пользовательского интерфейса.

## Требования к проекту
1. **Управление пользователями**
- Регистрация новых пользователей с уникальным логином
- Авторизация по логину и паролю
- Выход из аккаунта
- Смена пароля
2. **Социальные функции**
- Добавление пользователей в друзья
- Удаление из друзей
- Просмотр списка друзей с именами и ID
3. **Публикации**
- Создание постов на своей стене
- Просмотр своих публикаций
- Просмотр ленты новостей (посты друзей)
4. **Сообщения**
- Отправка личных сообщений другим пользователям
- Просмотр входящих сообщений
- Просмотр исходящих сообщений
- Автоматическая отметка о прочтении входящих сообщений
5. **Работа с файлами**
- Автоматическое сохранение базы данных в network_db.json
- Экспорт профиля пользователя в текстовый файл username_profile.txt

3. **Подробное описание классов**
3.1. *Класс `Content` (Абстрактный базовый класс)*
Атрибуты:
- `_id`: Уникальный идентификатор контента (UUID) — Принцип ООП: Инкапсуляция
- `_text`: Текстовое содержимое — Принцип ООП: Инкапсуляция
- `_author_id`: ID автора контента — Принцип ООП: Инкапсуляция
- `_timestamp`: Время создания (формат YYYY-MM-DD HH:MM:SS) — Принцип ООП: Инкапсуляция
Методы:
- `render()`: Возвращает отформатированное представление — Принцип ООП: Полиморфизм
- `to_dict()`: Сериализация в словарь для JSON — Принцип ООП: Полиморфизм
- `from_dict()`: Десериализация из словаря — Принцип ООП: Полиморфизм
Назначение: Базовый класс для всех типов контента. Не может быть создан напрямую (абстрактный).
3.2. *Класс `Post` (наследник Content)*
Атрибут: `_likes`: Количество лайков на публикации — Принцип ООП: Инкапсуляция
Методы:
- `like()`: Увеличивает счётчик лайков на 1
- `render()`: Переопределённый метод отображения — Принцип ООП: Полиморфизм
- `to_dict()`: Сериализация с полем "type": "Post" — Принцип ООП: Полиморфизм
Назначение: Представление публикации в ленте новостей.
3.3. *Класс `Message` (наследник Content)*
Атрибуты:
- `_receiver_id`: ID получателя сообщения — Принцип ООП: Инкапсуляция
- `_is_read`: Статус прочтения (по умолчанию False) — Принцип ООП: Инкапсуляция
Методы:
- `mark_as_read()`: Отмечает сообщение как прочитанное
- `render()`: Переопределённый метод со статусом — Принцип ООП: Полиморфизм
- `to_dict()`: Сериализация с полем `"type": "Message"` — Принцип ООП: Полиморфизм
Назначение: Представление личного сообщения между пользователями.
3.4. *Класс `User`*
Атрибуты:
- `__password` (Атрибут): Пароль (приватный, двойное подчёркивание) — Принцип ООП: Инкапсуляция
- `_friends_ids` (Атрибут): Список ID друзей — Принцип ООП: Инкапсуляция
- `_posts` (Атрибут): Список публикаций пользователя — Принцип ООП: Агрегация
- `_inbox` / `_outbox` (Атрибут): Входящие/исходящие сообщения — Принцип ООП: Агрегация
Методы:
- `check_password()`: Проверка пароля без возврата значения — Принцип ООП: Инкапсуляция
- `change_password()`: Смена пароля с проверкой старого
- `add_friend()`: Добавление друга с проверкой дубликатов
- `remove_friend()`: Удаление друга из списка
- `get_friends_ids()`: Возвращает копию списка (защита)
- `create_post()`: Создание и добавление публикации
- `send_message()`: Отправка сообщения (сохранение в outbox)
- `receive_message()`: Получение сообщения (добавление в inbox)
- `get_inbox()`: Возврат входящих с mark_as_read()
- `get_outbox()`: Возврат исходящих сообщений
- `export_profile_to_file()`: Экспорт в TXT файл — Работа с файлами
- `to_dict()` / `from_dict()`: Сериализация/десериализация
Назначение: Модель пользователя со всеми связанными данными.
3.5. *Класс `SocialNetwork`*
Атрибуты:
- `_users`: Словарь всех пользователей (ключ: user_id) — Принцип ООП: Инкапсуляция
- `_current_user`: Текущий авторизованный пользователь — Принцип ООП: Инкапсуляция
- `DB_FILE`: Имя файла базы данных (`network_db.json`)
Методы:
- `__init__()`: Инициализация с загрузкой БД
- `_load_db()`: Загрузка из JSON при старте — Работа с файлами
- `_save_db()`: Сохранение в JSON при изменениях — Работа с файлами
- `register()`: Регистрация нового пользователя
- `login()`: Авторизация по логину и паролю
- `logout()`: Сброс текущего пользователя
- `find_user_by_username()`: Поиск по логину
- `find_user_by_id()`: Поиск по UUID
- `get_current_user()`: Получение текущей сессии
Назначение: Класс-менеджер (Controller), управление всей сетью.

5. **Схема хранения данных**
*5.1. Файл: network_db.json*
*Формат: JSON*
*Содержимое*:
- id: uuid-1
- username: user1
- password: pass1
- friends_ids: ["uuid-2"]
- posts: [{"type": "Post", "text": "...", "likes": 0}]
- inbox: [{"type": "Message", "text": "...", "is_read": false}]
- outbox: [...]
*5.2. Файл: username_profile.txt*
*Формат: TXT*
*Содержимое*:
- Заголовок: === ПРОФИЛЬ ПОЛЬЗОВАТЕЛЯ: username ===
- ID: uuid-1
- Друзей (ID): 1
- Всего публикаций: 3
- Раздел: --- ПОСЛЕДНИЕ ПУБЛИКАЦИИ ---
- Раздел: --- ВХОДЯЩИЕ СООБЩЕНИЯ ---
**6. Выводы по анализу**
Архитектура
Паттерн MVC (Model-View-Controller) в упрощённом виде
Инкапсуляция
Приватные атрибуты, публичные методы доступа, возврат копий списков

Обработка ошибок
